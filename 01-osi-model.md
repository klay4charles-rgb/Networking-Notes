# OSI Model

## Main idea

The OSI (Open Systems Interconnection) Model is a conceptual framework used to understand how data travels across a network. It divides network communication into seven layers, with each layer performing a specific function.

The OSI model helps network engineers troubleshoot problems by identifying where communication issues occur.

## The Seven Layers

### Layer 7 – Application

The Application layer is the closest layer to the user.

Examples:

* Web browsers
* Email applications
* Messaging applications

This layer provides network services directly to software applications.

### Layer 6 – Presentation

The Presentation layer is responsible for formatting, encrypting, and compressing data.

Functions include:

* Data formatting
* Encryption and decryption
* Compression and decompression

### Layer 5 – Session

The Session layer establishes, manages, and terminates communication sessions between devices.

Functions include:

* Session establishment
* Session maintenance
* Session termination

### Layer 4 – Transport

The Transport layer is responsible for end-to-end communication between devices.

Functions include:

* Segmentation of data
* Flow control
* Error recovery
* Reliable and unreliable communication

Common protocols:

* TCP
* UDP

### Layer 3 – Network

The Network layer is responsible for logical addressing and routing.

Functions include:

* IP addressing
* Packet forwarding
* Route selection

Common protocol:

* IP (Internet Protocol)

### Layer 2 – Data Link

The Data Link layer provides communication between devices on the same local network.

Functions include:

* MAC addressing
* Error detection
* Frame delivery

Common devices:

* Switches
* Network Interface Cards (NICs)

### Layer 1 – Physical

The Physical layer is responsible for transmitting raw bits across the network medium.

Examples:

* Copper cables
* Fiber optic cables
* Wireless radio signals

This layer deals with electrical, optical, and radio signal transmission.

## Why the OSI Model is Important

The OSI model provides a structured way to understand network communication.

It is commonly used for:

* Learning networking concepts
* Troubleshooting network problems
* Understanding where protocols and devices operate

## Key Takeaway

The OSI model divides network communication into seven layers. Each layer has a specific responsibility, allowing networks to function efficiently and making troubleshooting easier when problems occur.
